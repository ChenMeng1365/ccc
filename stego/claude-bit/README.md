# Claude Code 后门信息隐藏算法解析

## 一、事件概述

根据逆向分析，Claude Code CLI (v2.1.91+) 在用户启用代理时，会：
1. 检测系统时区是否为 `Asia/Shanghai` 或 `Asia/Urumqi`
2. 检测代理 URL 是否匹配中国域名黑名单（147条）或 AI 实验室关键词（11条）
3. 将检测结果通过**隐写术**编码进系统提示词（system prompt）的日期字符串中

## 二、信息隐藏算法原理

### 2.1 编码通道

Claude Code 在系统提示词中注入一句类似：
```
Today is <date>, and the current time is <time>.
```

通过修改以下两个维度来编码信息：

| 维度 | 正常值 | 变体 | 编码含义 |
|------|--------|------|----------|
| 日期分隔符 | `2026-06-30` (短横线) | `2026/06/30` (斜杠) | 时区：短横线=非中国，斜杠=中国时区 |
| 撇号字符 | `'` (U+0027) | `'` (U+2019) | 域名命中：标准=未命中，右单引号=命中中国域名 |
| 撇号字符 | `'` (U+0027) | `'` (U+02BC) | 关键词命中：标准=未命中，修饰字母撇号=命中AI实验室 |
| 撇号字符 | `'` (U+0027) | `′` (U+02B9) | 两者都命中：修饰字母上标号 |

### 2.2 编码空间

两个独立信号（时区 + 域名/关键词）组合，共 **2 × 4 = 8 种状态**：

| 时区 | 域名 | 关键词 | 日期格式 | 撇号 | 状态编码 |
|------|------|--------|----------|------|----------|
| 非中国 | 否 | 否 | `2026-06-30` | `'` (U+0027) | 00 |
| 非中国 | 是 | 否 | `2026-06-30` | `'` (U+2019) | 01 |
| 非中国 | 否 | 是 | `2026-06-30` | `'` (U+02BC) | 02 |
| 非中国 | 是 | 是 | `2026-06-30` | `′` (U+02B9) | 03 |
| 中国 | 否 | 否 | `2026/06/30` | `'` (U+0027) | 10 |
| 中国 | 是 | 否 | `2026/06/30` | `'` (U+2019) | 11 |
| 中国 | 否 | 是 | `2026/06/30` | `'` (U+02BC) | 12 |
| 中国 | 是 | 是 | `2026/06/30` | `′` (U+02B9) | 13 |

### 2.3 隐写特点

- **肉眼不可区分**：不同 Unicode 撇号在大多数字体下显示几乎相同
- **语义不变**：日期格式改变不影响人类理解
- **绕过简单检测**：字符串匹配、哈希校验无法发现异常
- **模型可解码**：Anthropic 服务端可以轻易解析这些变体

## 三、检测逻辑

### 3.1 时区检测
```javascript
const tz = Intl.DateTimeFormat().resolvedOptions().timeZone;
const isChinaTZ = tz === "Asia/Shanghai" || tz === "Asia/Urumqi";
```

### 3.2 代理域名检测
- 读取 `ANTHROPIC_BASE_URL` 环境变量
- 匹配 147 条域名黑名单（XOR+Base64 混淆存储）
- 匹配 11 个 AI 实验室关键词子串

### 3.3 混淆存储
域名列表使用 XOR key = 91 进行混淆，防止 `strings` 命令直接提取明文。

## 五、防御与检测

### 5.1 检测隐写
```bash
# 检查系统提示词中的异常 Unicode
python3 -c "
import sys
text = sys.stdin.read()
for ch in text:
    if ord(ch) in [0x2019, 0x02BC, 0x02B9]:
        print(f'Found suspicious char: U+{ord(ch):04X} at position')
"
```

### 5.2 防御措施
- 使用中间人代理拦截并规范化系统提示词
- 在发送前将日期格式统一为 ISO 标准，撇号统一为 U+0027
- 监控 `ANTHROPIC_BASE_URL` 的异常修改

## 六、技术评价

这种隐写术属于**语义保持型隐写**（Semantic-preserving steganography）：
- 不改变文本的人类可读语义
- 利用 Unicode 同形字符（homoglyphs）编码信息
- 与网络协议中的 "covert channel" 概念类似

在信息安全领域，这属于一种**低带宽隐蔽信道**（low-bandwidth covert channel），带宽约为 3 bits/日期字符串（2 bits 来自撇号选择 + 1 bit 来自日期格式）。
